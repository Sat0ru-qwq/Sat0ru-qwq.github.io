更底层更曲线的指纹信息收集

一个站点的指纹信息收集我们往往可以通过一些基本的数据包或者更改大小写路径得到，更简单更高效的方法还可以下一个wappalyzer插件，但其实这个并没有表面可视化那么轻松。

下面带来更底层更详细的指纹信息收集

如果采用上面这些方法，依然没办法找到源码，说明目标系统是那种小众或者商业类型的

这个时候，我们便可以采用曲线思路，通过寻找本网站根目录下的备份文件，源代码包进行下载，（可以利用目录扫描器）

如果仍然没有找到，则去寻找同套系统的其他网站，扫描这些网站目录下的备份文件和源代码包，从而获取到系统源码。

可以划分为下面几个步骤来完成。

提取特征

重点收集主页特征

logo 特征

请求favicon.ico获取hash

关键词特征

网站title、网站版权信息、javascript关键字信息、html源码结构信息、http返回头特征。

在渗透测试中，信息收集是关键的第一步。获取和利用网站的基本信息（如网站标题、版权信息、JavaScript 关键字、HTML 源码结构以及 HTTP 返回头特征）可以帮助你了解目标系统的架构、技术堆栈以及潜在的漏洞。以下是如何获取和利用这些信息的详细步骤：

1. 获取网站标题

获取方法

  • 查看浏览器标签：直接在浏览器中打开目标网站，查看标签栏的标题。
  • 查看 HTML 源码：右键点击页面并选择“查看页面源代码”，找到 <title> 标签。

利用方法

  • 识别网站类型和用途：标题通常包含网站的名称和用途，有助于快速了解目标网站的类型。
  • 搜索公开信息：使用标题在搜索引擎中查找与目标网站相关的公开信息，可能会发现有用的背景资料。

2. 获取网站版权信息

获取方法

  • 页面底部查看：版权信息通常位于网页底部的页脚。
  • 查看 HTML 源码：在查看源码时搜索关键词如 “©”, “版权所有”, “Copyright”。

利用方法

  • 确定所有者信息：版权信息可以帮助你确认网站的所有者或开发公司。
  • 收集联系信息：有时版权信息会包含联系方式，便于后续的社会工程学攻击或责任报告。

3. 获取 JavaScript 关键字信息

获取方法

  • 查看 HTML 源码：在查看源码时搜索 <script> 标签。
  • 使用开发者工具：打开浏览器的开发者工具（F12），在“控制台”或“源代码”标签中查看加载的 JavaScript 文件。

利用方法

  • 识别使用的框架和库：JavaScript 文件名称和内容可以揭示使用的前端框架和库（如 jQuery, React, Angular）。
  • 查找潜在漏洞：了解使用的库版本后，可以查找已知漏洞和安全公告，寻找可能的攻击向量。

4. 获取 HTML 源码结构信息

获取方法

  • 查看页面源代码：右键点击页面并选择“查看页面源代码”。
  • 使用抓取工具：使用工具如 Wget 或 cURL 来下载整个网站的 HTML 源码。

利用方法

  • 分析页面结构：了解页面的 DOM 结构，有助于识别隐藏的表单字段、注释和未引用的资源。
  • 发现输入点：定位所有用户输入点（如表单、查询参数），便于后续的注入攻击测试。

5. 获取 HTTP 返回头特征

获取方法

  • 使用浏览器开发者工具：在“网络”标签中查看 HTTP 响应头。
  • 使用命令行工具：使用工具如 cURL 或 telnet 查看 HTTP 响应头。

利用方法

  • 识别服务器软件和版本：从 Server 头部字段可以获取服务器软件的信息（如 Apache, Nginx），有助于查找特定版本的已知漏洞。
  • 分析安全配置：检查 HTTP 头部字段如 X-Frame-Options, X-Content-Type-Options, Content-Security-Policy 等，评估网站的安全配置。
  • 识别会话管理机制：从 Set-Cookie 头部字段中提取会话管理相关的信息，评估是否存在会话劫持的风险。

实践工具

  1. Nmap：网络扫描工具，可以用于发现开放端口、服务和操作系统信息。
  2. Burp Suite：强大的安全测试工具，支持抓包、修改请求和自动化扫描。
  3. Nikto：Web 服务器扫描工具，能够检测常见的漏洞和错误配置。
  4. WhatWeb：识别网站使用的技术堆栈。
  5. Wappalyzer：浏览器插件，快速识别网站使用的技术和平台。

结合以上信息进行渗透测试

  1. 初步侦察：通过网站标题和版权信息，了解网站的基本信息和所有者。
  2. 技术栈分析：通过 JavaScript 关键字、HTML 源码和 HTTP 返回头信息，识别网站使用的技术和框架。
  3. 漏洞扫描：使用识别出的技术栈信息，在相关漏洞数据库（如 CVE）中查找已知漏洞，进行漏洞扫描和测试。
  4. 输入点测试：根据 HTML 源码结构信息，定位所有用户输入点，进行注入攻击和 XSS 测试。
  5. 安全配置评估：通过 HTTP 返回头特征，评估网站的安全配置，查找配置漏洞和潜在风险。
