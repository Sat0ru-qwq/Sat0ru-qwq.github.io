字符串

Go语言中，字符串使用了UTF8编码，UTF8的好处在于：如果基本是英文，每个字符占1byte，和ASCII编码是一样的，非常省空间。中文一般三个字节，包含中文的字符串处理方式和ASCII编码构成的字符串有点区别

  • reflect.TypeOf().Kind() 可以知道某个变量的类型，我们可以看到，字符串是以 byte 数组形式保存的，类型是 uint8，占1个 byte，打印时需要用 string 进行类型转换，否则打印的是编码值。
  • 因为字符串是以 byte 数组的形式存储的，所以，str2[2] 的值并不等于语。str2 的长度 len(str2) 也不是 4，而是 8（ Go 占 2 byte，语言占 6 byte）。

转换成 []rune 类型后，字符串中的每个字符，无论占多少个字节都用 int32 来表示，因而可以正确处理中文。

在 Go 语言中，处理中文字符串时使用 rune 类型主要是因为 Go 语言中的字符串实际上是一个字节序列，而中文字符通常由多个字节组成（UTF-8 编码）。为了正确处理和操作这些字符，需要将它们转换为 rune 类型。下面是具体的原因：

  1. UTF-8 编码：
    • Go 语言中的字符串是由 UTF-8 编码的字节构成的。在 UTF-8 编码中，一个字符可以由 1 到 4 个字节组成。例如，英文字符通常是 1 个字节，而中文字符通常是 3 个字节。
    • 因此，直接处理字符串中的字节可能导致问题，因为你无法确定每个字节是否完整地构成一个字符。
  2. rune 类型：
    • rune 是 Go 语言中用来表示 Unicode 字符的类型。它实际上是 int32 的别名，用来表示一个 Unicode 代码点。
    • 使用 rune 可以让你以字符为单位来处理字符串，而不是以字节为单位。这对于处理包含多字节字符的字符串非常重要。
  3. 迭代和处理字符：
    • 当你将字符串转换为 rune 数组（或切片）时，你可以逐个处理字符而不是字节。这使得字符串操作更直观，也避免了处理多字节字符时可能出现的问题。
    • 例如，使用 range 关键字遍历字符串时，Go 语言自动将其转换为 rune，这样你可以安全地处理每个字符。

本质原因：中文是多字节字符，处理时Go的逻辑将它单字符分解，因此无法实现对中文字符串的直接处理。

指针（Pointer)

指针即某个值的地址，类型定义时使用符号 *****

对一个已经存在的变量，使用&获取该变量的地址

一般来说，指针通常在函数传递参数，或者给某个类型定义新的方法时使用。Go 语言中，参数是按值传递的，如果不使用指针，函数内部将会拷贝一份参数的副本，对参数的修改并不会影响到外部变量的值。如果参数使用指针，对参数的传递将会影响到外部变量。

for range 的用法

for range 是 Go 语言中特有的一种语法，用于遍历各种数据结构。它在循环中自动提供了每个元素的索引和值，对于数组、切片、映射（map）、字符串等数据结构都适用。

  1. 遍历切片：

    在这个例子中，nums 是一个整数切片。for range 循环自动将切片中的每个元素的索引和对应的值赋给变量 i 和 num。i 是当前元素的索引，而 num 是当前元素的值。

  2. 遍历映射（map）：

    对于映射 m2，for range 循环自动提供了每对键值对的 key 和 value。这里 key 是映射中的键，value 是对应的值。

为什么 Go 能自动识别 key 和 value

Go 语言通过 for range 语法实现了对不同数据结构的遍历，具体的识别过程如下：

  • 数组和切片： for range 遍历时，Go 会自动提供当前元素的索引和对应的值。对于数组和切片，range 生成一个索引和元素值的对。
  • 映射（map）： for range 遍历时，Go 会自动提供当前键值对的键和值。对于映射，range 生成一个键和值的对。
  • 字符串： for range 遍历时，Go 会自动提供每个字符的索引和字符值（以 rune 类型表示）。
总结
  • for 循环 用于基本的迭代需求，像你最初的整数循环示例。
  • for range 用于更复杂的遍历需求，它能够自动提供每个元素的索引（对于切片和数组）或键值对（对于映射）。

for range 使得遍历集合类型（如切片、映射、字符串）变得简单且直观，同时通过 i, num 和 key, value 的方式自动处理了元素的解包和赋值。
